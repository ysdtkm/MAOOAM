30 ## Installation ##
31 
32 The program can be installed with Makefile. We provide configuration files for
33 two compilers : gfortran and ifort.
34 
35 By default, gfortran is selected. To select one or the other, simply modify the
36 Makefile accordingly. If gfortran is selected, the code should be compiled
37 with gfortran 4.7+ (allows for allocatable arrays in namelists).
38 If ifort is selected, the code has been tested with the version 14.0.2 and we
39 do not guarantee compatibility with older compiler version.
40 
41 To install, unpack the archive in a folder, and run:
42  make
43 
44  Remark: The command "make clean" removes the compiled files.
45 
46 For Windows users, a minimalistic GNU development environment
47  (including gfortran and make) is available at [www.mingw.org](http://www.mingw.org) .

51 ## Description of the files ##
52 
53 The model tendencies are represented through a tensor called aotensor which
54 includes all the coefficients. This tensor is computed once at the program
55 initialization.
56 
57 * maooam.f90 : Main program.
58 * aotensor_def.f90 : Tensor aotensor computation module.
59 * IC_def.f90 : A module which loads the user specified initial condition.
60 * inprod_analytic.f90 : Inner products computation module.
61 * rk2_integrator.f90 : A module which contains the Heun integrator for the model equations.
62 * rk4_integrator.f90 : A module which contains the RK4 integrator for the model equations.
63 * Makefile : The Makefile.
64 * gfortran.mk : Gfortran compiler options file.
65 * ifort.mk : Ifort compiler options file.
66 * params.f90 : The model parameters module.
67 * tl_ad_tensor.f90 : Tangent Linear (TL) and Adjoint (AD) model tensors definition module
68 * rk2_tl_ad_integrator.f90 : Heun Tangent Linear (TL) and Adjoint (AD) model integrators module
69 * rk4_tl_ad_integrator.f90 : RK4 Tangent Linear (TL) and Adjoint (AD) model integrators module
70 * test_tl_ad.f90 : Tests for the Tangent Linear (TL) and Adjoint (AD) model versions
71 * README.md : A read me file.
72 * LICENSE.txt : The license text of the program.
73 * util.f90 : A module with various useful functions.
74 * tensor.f90 : Tensor utility module.
75 * stat.f90 : A module for statistic accumulation.
76 * params.nml : A namelist to specify the model parameters.
77 * int_params.nml : A namelist to specify the integration parameters.
78 * modeselection.nml : A namelist to specify which spectral decomposition will be used.

83 ## Usage ##
84 
85 The user first has to fill the params.nml and int_params.nml namelist files according to their needs.
86 Indeed, model and integration parameters can be specified respectively in the params.nml and int_params.nml namelist files. Some examples related to already published article are available in the params folder.
87 
88 The modeselection.nml namelist can then be filled :
89 * NBOC and NBATM specify the number of blocks that will be used in respectively the ocean and
90  the atmosphere. Each block corresponds to a given x and y wavenumber.
91 * The OMS and AMS arrays are integer arrays which specify which wavenumbers of
92  the spectral decomposition will be used in respectively the ocean and the
93  atmosphere. Their shapes are OMS(NBOC,2) and AMS(NBATM,2).
94 * The first dimension specifies the number attributed by the user to the block and the second
95  dimension specifies the x and the y wavenumbers.
96 * The VDDG model, described in Vannitsem et al. (2015) is given as an example
97  in the archive.
98 * Note that the variables of the model are numbered according to the chosen
99  order of the blocks.
100 
101 
102 Finally, the IC.nml file specifying the initial condition should be defined. To
103 obtain an example of this configuration file corresponding to the model you
104 have previously defined, simply delete the current IC.nml file (if it exists)
105 and run the program :
106 
107  ./maooam
108 
109 It will generate a new one and start with the 0 initial condition. If you want another
110 initial condition, stop the program, fill the newly generated file and restart :
111 
112  ./maooam
113 
114 It will generate two files :
115  * evol_field.dat : the recorded time evolution of the variables.
116  * mean_field.dat : the mean field (the climatology)
117 
118 The tangent linear and adjoint models of MAOOAM are provided in the
119 tl_ad_tensor, rk2_tl_ad_integrator and rk4_tl_ad_integrator modules. It is
120 documented [here](./md_tl_ad_doc.html).

125 ## Implementation notes ##
126 
127 As the system of differential equations is at most bilinear in \f$y_j\f$ (\f$j=1..n\f$),
128 \f$\boldsymbol{y}\f$ being the array of variables, it can be expressed as a tensor
129  contraction :
130 
131 \f[ \frac{d y_i}{dt} = \sum_{j,k=0}^{ndim} \, \mathcal{T}_{i,j,k} \, y_k \; y_j \f]
132 
133 with \f$y_0 = 1\f$.
134 
135 
136 The tensor aotensor_def::aotensor is the tensor \f$\mathcal{T}\f$ that encodes the differential equations is composed so that:
137 
138 * \f$\mathcal{T}_{i,j,k}\f$ contains the contribution of \f$dy_i/dt\f$ proportional to \f$ y_j \, y_k\f$.
139 * Furthermore, \f$y_0\f$ is always equal to 1, so that \f$\mathcal{T}_{i,0,0}\f$ is the constant
140 contribution to \f$dy_i/dt\f$
141 * \f$\mathcal{T}_{i,j,0} + \mathcal{T}_{i,0,j}\f$ is the contribution to \f$dy_i/dt\f$ which is linear in
142 \f$y_j\f$.
143 
144 Ideally, the tensor aotensor_def::aotensor is composed as an upper triangular matrix
145 (in the last two coordinates).
146 
147 The tensor for this model is composed in the aotensor_def module and uses the
148 inner products defined in the inprod_analytic module.
